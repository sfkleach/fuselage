## Background: Herescript

Repo: https://github.com/sfkleach/herescript
README: https://raw.githubusercontent.com/sfkleach/herescript/refs/heads/main/README.md

## Format

The format is a base64 encoding of either of the archival formats: zip or
squashfs. However it may be preceded by an arbitrary number of comment lines
which must begin with `#` in column 1.

## Step 1: Recognise base64 (with comments)

Extend --static and --dynamic so that they recognise that the supplied file
is neither of the archival formats, then attempt to decode the file in the 
tmpfs (so that it does not leak out). 

If this is successful then the unpacked file is checked for being a zip or
squashfs. If it passes that test then the normal logic takes over. If
--cache-static is enabled the unpacked file is cached.

## Step 2: Construct an integration test with herescript

Write a test script that:

- Creates a folder that contains a 'hello world' program called `hello.sh` (chmod +x).
- Converts it to sfs format.
- Converts that to base64.
- Then pastes it onto the end of a shebang template that looks
  like:

```
#!herescript fuselage
#: --static testapp:${HERESCRIPT_FILE} 
#: --run myapp/hello.sh
<BASE 64 CONTENT GOES HERE>
```

Then it runs the test-script and verifies its output.
